# Hermes Sync — Cross-Device Configuration Sync

Sync your Hermes Agent settings (config, skills, profiles, cron jobs) across
multiple machines using a private GitHub repo.

**When to use this:** You use Hermes on more than one machine and want
identical config on all of them.

---

## Key Paths (Windows vs Unix)

On **Windows Hermes Desktop**, config lives at:
```
C:\Users\<user>\AppData\Local\hermes
```

On **Unix/macOS** or **WSL**:
```
~/.hermes
```

Always use these commands to find the actual paths:
```bash
hermes config path      # prints config.yaml location
hermes config env-path  # prints .env location
```

For this skill, the base path is referenced as `$HERMES_HOME` — substitute
the Windows or Unix path above as appropriate.

---

## What to Sync vs. Keep Local

### Sync (commit to git)
- `config.yaml` — model, provider, tools, terminal settings
- `skills/` — custom + hub-installed skills
- `profiles/` — named profiles
- `cron/` — scheduled jobs (but exclude lock files)
- `SOUL.md` — agent identity

### NOT Sync (never commit)
- `.env` — API keys and secrets
- `auth.json` — OAuth tokens (device-scoped)
- `logs/` — log files
- `sessions/` — session history (device-specific paths)
- `state.db` — SQLite session store
- `cache/` — model catalogs, caches
- `bin/`, `shared/`, `node/` — Hermes bundled binaries
- `hermes-agent/` — Hermes source code
- `zadon`, `zadon.pub` — SSH key files generated by Hermes

## Desktop A — Initial Setup

### 1. Create a Private GitHub Repo

1. Go to https://github.com/new
2. Repository name: `hermes-config`
3. Visibility: **Private**
4. Do NOT initialize with README or .gitignore

### 2. Add SSH Key to GitHub

Generate a new SSH key (if not already present):
```bash
ssh-keygen -t ed25519 -C "hermes-sync" -f ~/.ssh/id_ed25519 -N ""
```

Add the public key to GitHub:
- Go to https://github.com/settings/keys
- Click "New SSH key"
- Title: `My Desktop` (or any name)
- Key: paste contents of `~/.ssh/id_ed25519.pub`

Verify connection:
```bash
ssh -T git@github.com
# Expected: "Hi <username>! You've successfully authenticated..."
```

### 3. Initialize Git in Hermes Config Directory

```bash
cd "$HERMES_HOME"
git init
git remote add origin git@github.com:YOUR_USERNAME/hermes-config.git
```

Set git identity (required before first commit):
```bash
git config user.email "your@email.com"
git config user.name "Your Name"
```

### 4. Create `.gitignore`

This is critical — prevents secrets and large binaries from being committed:

```gitignore
# Secrets - NEVER commit
.env
.env.*
auth.json
auth.json.*
auth.lock

# Database
state.db
state.db-*
state.db-shm
state.db-wal

# Hermes bundled binaries (large - don't track)
bin/
shared/
git/
node/
node_modules/
cache/
bootstrap-cache/

# Logs and sessions
logs/
sessions/
*.log

# Cron lock files
cron/.jobs.lock
cron/.tick.lock
cron/ticker_*
cron/heartbeat
cron/last_success

# State snapshots
state-snapshots/

# Audio/image cache
audio_cache/
image_cache/
memories/

# Pairing
pairing/

# Hermes source
hermes-agent/

# OS files
.DS_Store
Thumbs.db

# Hermes installer
desktop-build-stamp.json
hermes-setup.exe

# Misc
zadon
zadon.pub
.skills_prompt_snapshot.json
.update_check
.update_exit_code
ollama_cloud_models_cache.json
models_dev_cache.json
```

### 5. First Commit and Push

```bash
cd "$HERMES_HOME"
git add .gitignore config.yaml SOUL.md skills/
git commit -m "Initial Hermes config sync"
git branch -M main
git push -u origin main
```

### 6. Back Up `.env`

The `.env` file contains API keys. Copy it to a password manager
(1Password, Bitwarden, etc.) — you'll need it for Desktop B.

```bash
cat "$HERMES_HOME/.env"
```

### 7. Create the Sync Helper Script

```bash
mkdir -p ~/.local/bin

# For Windows (git-bash):
cat > ~/.local/bin/hsync <<'EOF'
#!/usr/bin/env bash
cd /c/Users/YOUR_USERNAME/AppData/Local/hermes || {
    echo "Hermes config not found"
    exit 1
}
git add -A
git commit -m "sync: $(date '+%Y-%m-%d %H:%M')" 2>/dev/null || true
git pull --rebase origin main
git push origin main
echo "Sync complete"
EOF

# For Unix/macOS:
cat > ~/.local/bin/hsync <<'EOF'
#!/usr/bin/env bash
cd ~/.hermes || { echo "Hermes config not found"; exit 1; }
git add -A
git commit -m "sync: $(date '+%Y-%m-%d %H:%M')" 2>/dev/null || true
git pull --rebase origin main
git push origin main
echo "Sync complete"
EOF

chmod +x ~/.local/bin/hsync
```

Add to PATH if needed:
```bash
# bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc

# zsh
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
```

## Desktop B — Clone and Setup

### 1. Install Hermes

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

### 2. Clone Config from GitHub

```bash
cd "$HERMES_HOME"
git init
git remote add origin git@github.com:YOUR_USERNAME/hermes-config.git
git pull origin main
```

If there are conflicts with existing files:
```bash
rm -rf auth.json logs sessions state.db
git pull origin main
```

### 3. Recreate `.env`

Get the content from your password manager (Desktop A, Step 6), then:
```bash
cat > "$HERMES_HOME/.env" <<'EOF'
# Paste your .env content here
...
EOF
```

### 4. Re-authenticate OAuth Providers

```bash
hermes auth
```

Some providers (GitHub Copilot, OpenAI Codex, Nous Portal) require
interactive login on each device.

### 5. Install the Sync Helper

Same as Desktop A, Step 7 — copy the `hsync` script to `~/.local/bin/`.

---

## Daily Usage

Run `hsync` before and after each session:

```bash
hsync
```

This will:
1. Stage all changes
2. Commit with a timestamp
3. Pull any changes from the other machine
4. Push back up

For automatic sync, create a cron job:
```bash
hermes cron create "45 9,18 * * *" \
  --prompt "Run: hsync" \
  --name "Daily Hermes sync"
```

## Troubleshooting

### "Permission denied" when pushing
- SSH key not added to GitHub → see Step 2 above
- Wrong SSH key being used → check `~/.ssh/config` or run:
  ```bash
  ssh -vT git@github.com
  ```

### Large file warnings on push
- Some large files (e.g. `bin/uv.exe` at 66MB) may exceed GitHub's 50MB limit
- They should already be in `.gitignore` — if not, add them and push again

### OAuth tokens don't work on Desktop B
- OAuth tokens (GitHub Copilot, etc.) are device-scoped by design
- Run `hermes auth` on Desktop B and re-authenticate

### Git identity error on commit
```bash
git config user.email "your@email.com"
git config user.name "Your Name"
```

### SSH key generation on Windows hangs
- Use `-N ""` to set empty passphrase and skip interactive prompts:
  ```bash
  ssh-keygen -t ed25519 -C "hermes-sync" -f ~/.ssh/id_ed25519 -N ""
  ```

## Alternative: Profile Export/Import

If you don't want a full git repo, Hermes has built-in export:
```bash
# Desktop A
hermes profile export default --output ~/hermes-profile.tar.gz

# Desktop B
hermes profile import ~/hermes-profile.tar.gz
```

This handles config + skills + profiles in one file, but:
- It's a one-time transfer, not continuous sync
- You still need to recreate `.env` and re-auth OAuth manually
- Git sync is better for ongoing use
